About This Project

This website is intentionally designed to be **vulnerable** for learning and testing security concepts. It allows users to practice identifying and exploiting security flaws while understanding how to patch them.

🔍 Project Goals

  • Teach **OWASP Top 10 vulnerabilities** in a hands-on manner.
  • Demonstrate **SQL Injection**, **Broken Access Control**, and **Security Misconfigurations**.
  • Help security researchers and developers practice **penetration testing**.

⚠️ Known Vulnerabilities

  • 🚨 **SQL Injection** (Login page vulnerable to direct query manipulation).
  • 🔓 **Broken Access Control** (Anyone can access `/admin`).
  • 📂 **Security Misconfiguration** (Exposing error messages in responses).

🛠 Future Fixes

  • Use **prepared statements** to prevent SQL Injection.
  • Implement **authentication & session management**.
  • Remove **exposed error messages**.

📚 This project is for educational purposes only. Please do not use these techniques on unauthorized systems.
